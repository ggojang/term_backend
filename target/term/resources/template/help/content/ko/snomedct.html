1. 상단메뉴

  • 인포클리닉 로고

  • 브라우저 : SNOMED CT 브라우저/LOINC 브라우저 선택. 선택된 상태에서는 브라우저 초기화

  • 버전 : InfoClinic SNOMED CT Browser에 적용된 SNOMED CT 버전 표시 (예. International Edition 2017.01.31)

  • 언어선택 : 사용자가 사용 중인 언어에 따라 한글/영어 선택 (기본값은 사용 중인 브라우저 언어 세팅에 따름) (예. 한국어)

2. 탐색

1) 코드/문자열 검색

  • SNOMED CT 코드(SNOMED CT ID, SCTID) 또는 사람들이 읽을 수 있는 SNOMED CT 용어(Description)로 검색

  • SNOMED CT 용어는 Fully Specified Name(완전하게 기술된 명칭, FSN, F), Preferred Term (선호용어, PT, P), Synonym(동의어, S)로 구분되지만, 대부분의 FSN은 Preferred Term와 같고, Preferred Term도 Synonym 중에 하나이기 때문에 검색결과에는 Preferred Term은 제외한 FSN과 Synonym이 출력됨.

  • 검색하려는 SNOMED CT 용어는 영문 기준으로 대/소문자 구분이 없으며, 최소 2글자 이상이어야 함

    • 주) 검색란이 비어있을 때는 “ 검색결과” 영역에 SNOMED CT의 계층구조가 보여짐

2) 일치형태

  • 완전 일치 : 검색하는 문자열과 똑같이 일치하는 SNOMED CT 용어 검색
  • 부분 일치 : 검색하는 문자열이 포함된 SNOMED CT 용어 검색 (기본 선택)
  • 정규식 : Elastic Search에서 제공하는 정규식 지원 (ElasticSearch's Regular Expression)

3) 활성상태

  • 활성 : SNOMED CT 컨셉(Concept) 중 활성 (active) 상태인 컨셉들을 검색 (기본 선택)
  • 비활성 : SNOMED CT 컨셉 중 비활성 (inactive) 상태인 컨셉들을 검색
  • 활성 & 비활성 : SNOMED CT 컨셉 중 활성 및 비활성 상태인 컨셉 모두를 검색

4) 검색결과

검색란 아래에 검색된 갯수와 시맨틱태그 출력

  • 시맨틱태그 : 검색된 결과에 포함된 시맨틱태그의 갯수를 출력하며, 버튼 클릭시 원하는 시맨틱태그를 선택하여 출력함

  • 좌측 : 검색한 문자열과 일치하는 모든 SNOMED CT 용어(FSN과 synonym에 상관없이) 출력

  • 우측 : 좌측에 출력된 SNOMED CT 용어의 FSN을 출력

  • 용어 클릭 : 검색의 결과인 용어를 클릭하면 해당 용어의 컨셉을 중간 “컨셉 상세내용” 공간에 출력

용어 표기법

  • : Fully Specified Name (FSN)
  • : Synonym
  • : Fully defined concept
  • : primitive concept

5) 시맨틱 태그

  • 검색된 결과와 관련된 시맨틱태그를 출력
  • 각 시맨틱태그에 속하는 검색결과 개수를 출력
  • 사용자가 원하는 시맨틱태그를 선택할 수 있음 (중복선택가능)

6) 계층구조

"코드/문자열 검색" 부분이 비어있는 경우, SNOMED CT의 다중계층을 트리형태로 출력함

  • ">" : 트리 펼침
  • "v" : 트리 접음

3. 개요

화면 중앙에 선택된 용어의 상세내용을 출력함.

  • Stated : SNOMED International의 SNOMED CT 컨텐트 저작자들이 작성한 정의

  • Inferred : Stated를 기반으로 추론된 정의 (기본 선택)

1) 정의

SNOMED CT는 기술로직(Description Logic)에 따라 용어를 정의한다.

  • 정의(Defining Relationship) : 어떤 용어를 정의하는 방식은 바로 상위에 위치한 부모(direct parent, IS-A relationship)와 부모가 가지는 속성관계보다 상세화된(refined) 속성관계(attribute relationship)로 정의됨

2) 다이어그램

  • SNOMED International에서 정의한 방식으로 정의를 다이어그램 형태로 출력 (PNG 형태로 저장 가능)

3) 후조합 표현식

  • SNOMED CT 컨셉의 정의를 SNOMED CT Composition Grammar로 출력

4) 용어

언어별로 SNOMED CT 용어를 출력 (영어 기본)



1) DescriptionId : 해당 용어에 대한 유일한 식별자(ID)

2) Type : SNOMED CT에서 정의한 용어의 타입
  • F : Fully Specified Name
  • P : Preferred Term
  • S : Synonym
3) Term : 사람을 읽을 수 있는 형태의 용어 문자열

4) Acceptabiity : SNOMED CT Description에 포함된 용어 중 실제로 사용할 용어들을 지정하고 타입을 구분
  • : 동의어(Synonym) 중에서 FSN으로 지정
  • : 동의어(Synonym) 중에서 PT로 지정
  • : 동의어(Synonym)로 지정

5) 레퍼런스 세트

SNOMED International에서 배포하는 SNOMED CT International Edition에 포함된 Reference Set의 정보를 출력

  • CTV3/SNOMED RT : Simple Map Type Reference Set
  • ICD-9CM : Complex Map Type Reference Set
  • ICD-10 : Extend Map Type Reference Set

6) 이력

조회된 SNOMED CT 컨셉 및 용어에 변경사항에 대한 이력을 표시함

4. 부모/자식컨셉 및 수식어

화면 우측탭에 선택된 컨셉과 관련된 부모 및 자식 컨셉과 선택된 컨셉 정의에 따라 수식어를 출력

1) 부모컨셉

“상세내용”에 출력된 컨셉의 부모 컨셉을 출력

  • “>” : 트리 펼침
  • “V” : 트리 접음

2) 자식컨셉

“컨셉 상세내용”에 출력된 컨셉의 자식 컨셉을 출력

  • “>” : 트리 펼침
  • “V” : 트리 접음

3) 수식어

  • “개요”에 출력된 컨셉 정의 부분에서 하나의 그룹을 선택하면 컨셉의 의미론적 분류(Semantic Tag)에 따라 정의된 영역(Domain)과 범위(Range) 관계 출력

  • SNOMED CT는 컨셉 정의에 사용되는 Attribute (예. Associated morphology, finding site 등) 외에 수식어를 추가하여 사용할 수 있는 방법은 제공하고 있으며, SNOMED CT의 컨셉은 컨셉의 의미론적 분류(Semantic Tag)에 따라 사용할 수 있는 수식어가 영역(Domain)으로 정해져 있으며 해당 영역의 값은 또한 범위(Range)로 정해져 있음
