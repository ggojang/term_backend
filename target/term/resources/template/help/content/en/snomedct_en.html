1. Top Menu

  • InfoClinic Logo

  • Browser : Choose SNOMED CT Browser/LOINC Browser. Initiate Browser if it already is choosed.

  • Version : SNOMED CT Edition applied to InfoClinic SNOMED CT Browser (e.g International Edition 2017.01.31)

  • Langauge Selection : Select user language (Hanguel/English) (e.g English)

2. Explore

1) Code/Text Search

  • Search with SNOMED CT code(SNOMED CT ID, SCTID) or Human-readable SNOMED CT Description

  • SNOMED CT Descriptions are separated with Fully Specified Name(FSN, F), Preferred Term(PT, P), Synonym(S) but most FSNs are the same to Preferred Terms and Preferred Terms are also one of Synonyms so results are displayed with FSNs and Synonyms without Preferred terms.

  • Description for search is case-insentive and has at lease 2 characters.

    • note) If search area is empty, SNOMED CT polyhierarchy structure is displayed.

2) Match

  • Full text matching : Search for SNOMED CT Descriptions that match exactly the string you are looking for
  • Partial matching : Search for SNOMED CT Descriptions that partially match the string (default)
  • Regular Expressions : Support Elastic Search regular expression (ElasticSearch's Regular Expression)

3) Status

  • Active : Search SNOMED CT Concepts that are active (default)
  • Inactive : Search SNOMED CT Concepts that are inactive
  • Active and Inactive : Search SNOMED CT Concepts that are active and inactive, i.e all concepts

4) Search Results

Display the number and semantic tag results below search area.

  • Semantic Tag : Display the number of semantic tags included in the retrieved result, and selects the desired semantic tag when the button is clicked.

  • Left-side : Display all SNOMED CT terms (regardless of FSN and synonym) that match the retrieved string

  • Right-side : display the FSN of the SNOMED CT descriptions on the left side

  • Click Description : Click on a Description that is the result of a search and display the concept of the description in the middle "DETAIL" area

Notation

  • : Fully Specified Name (FSN)
  • : Synonym
  • : Fully defined concept
  • : primitive concept

5) Semantic Tag

  • display the semantic tag associated with the retrieved result
  • display the number of search results belonging to each semantic tag
  • Users can select the desired semantic tag (duplicate selection possible)

6) Hierarchical Structure

If the "Code / Text Search" part is empty, display the SNOMED CT polyhierarchy as a tree

  • “>” : Tree unfold
  • “V” : Tree fold

3. Overview

Display the details of the selected term in the center of the screen.

  • Stated : Definitions written by content authors in SNOMED CT International

  • Inferred : Inferred definition based on Stated (default)

1) Defining Relationship

SNOMED CT defines Concepts according to the description logic.

  • Defining Relationship : The definition of a Concept is defined as parents(direct parent, IS-A relationship) and Attribute Relationship that is more refined than a parent's.

2) Diagram

  • Display Defining Relationship in the form defined by SNOMED International in diagram form (Users can be saved in PNG format)

3) Post-Coordinated Expression

  • Display Defining Relationship to SNOMED CT Composition Grammar

4) Description

Display SNOMED CT Descriptions by language (Default is English)



1) DescriptionId : Unique Identifier of the Description(ID)

2) Type : Description Type defined by SNOMED CT
  • F : Fully Specified Name
  • P : Preferred Term
  • S : Synonym
3) Term : Human-readable strings

4) Acceptabiity : Specify Descriptions Type to be actually used among the terms included in SNOMED CT Description
  • : Specify FSN among Synonyms
  • : Specify PT among Synonyms
  • : Specify Synonym

5) Reference Set

Display the contents of Reference Set distributed by SNOMED International

  • CTV3/SNOMED RT : Simple Map Type Reference Set
  • ICD-9CM : Complex Map Type Reference Set
  • ICD-10 : Extend Map Type Reference Set

6) History

Displays the History of changes in the SNOMED CT Concept and Descriptions viewed

4. Parent/Children Concept & Qualifier

Display Qualifier and Parent/Children Concept in the tab on the right side of the screen according to selection from Attribute Relationship of Defining Relationship

1) Parent Concept

Display Parent Concept of selected Concept in the "DETAIL" "area.

  • “>” : Tree unfold
  • “V” : Tree fold

2) Children Concept

Display Children Concept of selected Concept in the "DETAIL' area.

  • “>” : Tree unfold
  • “V” : Tree fold

3) Qualifier

  • Selecting a Attribute Relationship in the Defining Relationship output in the "OVERVIEW" area and displays the Domain and Range relation according to the concept's semantic tag

  • SNOMED CT provides a method to add qualifiers besides the attributes (eg, associated morphology, finding site, etc.) used in Defining Relationship but the concept of SNOMED CT is classified to Semantic Tag and Semantic Tag has defined Domain and Range (e.g. Finding Site(Domain) : Appendix structure (anatomical site(Range)).
